# issue pullableと表示されていたがpullableでないことがあった #84
[issues #84](https://github.com/cat2151/cat-repo-auditor/issues/84)


- これは「新しい操作を増やす機能追加」というより、既存の `pullable` 判定を実際の `git pull --ff-only` 成否に近づけるための **判定改善** である。

- これまでの判定
  - local と upstream の commit 関係だけを見て、fast-forward 可能なら `pullable` と表示していた。
  - そのため、working tree に未解決 conflict が残っていて `git pull` が失敗する状態でも、`pullable` と表示されることがあった。

- 変更後の判定
  - まず unmerged files の有無を確認する。
  - unmerged files がある場合は、commit 関係上は fast-forward 可能でも `pullable` とは表示しない。
  - unmerged files がなく、従来どおり fast-forward 可能な場合のみ `pullable` と表示する。

- before / after
  - before
    - `git pull` 実行前に conflict 状態を見分けられず、`pullable` と表示される場合があった。
    - `auto_pull` が有効な場合も、その repo が pull 対象として扱われうる。
  - after
    - conflict 状態の repo は事前に `pullable` から除外される。
    - user から見ると、「pull できる repo」と「履歴上は追従可能だが、conflict 解消が先な repo」が区別される。
    - `auto_pull` でも、unmerged files がある repo は pull 対象に入らなくなる。
