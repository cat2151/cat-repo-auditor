# issue cargo チェック方式を修正する #51
[issues #51](https://github.com/cat2151/cat-repo-auditor/issues/51)

- userがほしいもの
    - 「install済みのcommit hash」の取得方法を修正する：
        - commit hash取得方法：
            - app name取得
                - `.crates2.json` をパースし、
                - 例えば、
                - `"cat-repo-auditor 0.1.0 (git+https://github.com/cat2151/cat-repo-auditor#<ランダム数値A>)":{"version_req":null,`
                - から、
                - app name:
                    - `cat-repo-auditor`
                - を得る
                - ※なお、このとき、`<ランダム数値A>`は、実はinstall時のcommit hash「ではない」ことが判明しているため、利用しない
            - cargo配下の、`git\checkouts\<app name><ランダム文字列B>\`にて、`git rev-parse HEAD` コマンドを実行してinstall時のcommit hashを得る
                - `git\checkouts\`のdir群をlistし、app nameで前方一致検索をする
                    - 複数hitした場合はエラーログをlocal config logs/log.txt に追記し、cargoチェック結果は「？」を表示とする
    - 参考
        - 既存アプリのvoicevox-playground-tuiにおいては、セルフアップデートのために、build時のcommit hashをセルフアップデート処理に埋め込みをしていた（ビルドツールが環境変数生成し、それをソースが参照していた）
        - これはアプリそれぞれが採用するには有用である、と判断する
        - これを「cat-repo-auditor側が利用」はできない。
            - これはあくまでアプリそれぞれのなかで閉じたものであるので。
                - もし利用しようとして例えば`--commit-hash`でcommit hashを得るようにアプリ側の共通規格を設計、などをするなら、それは有用性が低い。なぜなら変化に弱い。
                    - もっとよいのが、前述の、checkoutsを参照する方法である、と判断する
